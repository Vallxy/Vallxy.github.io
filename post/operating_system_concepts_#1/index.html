2.2.1 Command Interpreters

操作系统把它当作是一个在进程初始化或者用户第一次登陆交互式系统时运行的一个特殊程序

shells

shell命令的两种实现方法：

  1. command interpreter对于每一个命令有自己的实现代码，添加一条需要调整command interpreter

  2. 通过system program实现，command interpreter只控制将文件load进memory和executed (UNIX就是这种)，这种方法增加命令只需要创建新文件就可以了，不用动command interpreter

    Example：

    rm file.txt

    command interpreter要去找一个叫rm的文件，将其load入memory，然后带参数 file.txt 执行rm

2.3 System calls

  • system calls 提供了操作系统向外提供服务接口
2.3.2 Application Programming Interface

  • Even simple programs may make heavy use of the operation system.

  • API

  • A programmer access an API via a library of code provided by the operation system. In the case of UNIX and Linux for programs written in the C language, the library is called libc.

  • Each operating system has its own name for each system call.

  • Why programming according to API rather than invoking actual system calls?

    1. program portability
    2. Actual system calls————–detailed, difficult to work with

  • run-time environment (RTE)

    RTE provides a system-call-interface that servers as the link to system calls.

    The system-call interface intercepts function calls in the API and invokes the necessary system calls within the operating system.

    code —-> API —-> system-call-interface —-> system call

  • Most detailed interface are hidden from the programmer by the API and are managed by the RTE.

  • 3 general methods to pass paramaters to the operating system

    1. pass the parameters in register

    2. When parameters are too many, they are stored ina block, or table, in memory, and the address of the block is passed as parameter in a register.

    3. combination of 1 and 2 (Linux)

      if parameter number > 5:

      ​ use 2

      else:

      ​ use 1

2.5 Linkers and Loaders

  • source program —-> complier —-> object file (with other object files) —-> linker —->executable file (with dynamically linked libraries) —-> loader —-> program in memory

  • Why use DLLs?

    avoid linking and loading libraries that may end up not being used into an executable file

  • Executable and Linkable Format (ELF) in UNIX and Linux

2.6 Why Applications Are Operating-System Specific

  • applications compiled on one operating system are not executable on other operating systems

  • each operating system provides a unique set of system calls

  • 3 ways for an application available to run on multiple operating systems

    1. written in interpreted language (Python, Ruby)
    2. written in languages that includes a virtual machine
    3. use a standard language or API in which the compiler generates binaries in a machine- and operating-system-specified language. Then the application will be ported to different OS.

  • application level ——– API

    architecture level ——– application binary interface (ABI) : low-level details
